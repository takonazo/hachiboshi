謎解き用 架空製薬会社サイト 要件書
====================================

1. 目的
------
本サイトは、謎解き・ARG的な体験に使用する架空の製薬会社ウェブサイトである。
通常の企業サイトとして自然に見えることを前提にしつつ、参加者がサイト内の情報、ニュース記事、社内報、検索結果などを読み解くことで、次の手がかりに到達できる構成にする。

2. サイト全体の方針
------------------
- 架空の製薬会社・ヘルスケア関連企業の公式サイトとして見えるデザインにする。
- トップページは縦に長いLP風の1ページ構成にする。
- ただし、すべての情報をトップページに載せるのではなく、一部の情報は別ページに分ける。
- 通常のナビゲーションで到達できるページと、サイト内検索でしか到達できないページを用意する。
- 謎解きに必要な情報は、企業サイトとして不自然になりすぎない形で配置する。

3. 想定ユーザー
--------------
- 謎解きの参加者。
- 最初は一般的な企業サイトとして閲覧する。
- ニュース、社員紹介、検索機能などを調べながら、隠された情報や次の導線を探す。

4. トップページ要件
------------------
トップページは、企業サイトらしいメインページとして作成する。

必須要素:
- ページ上部に大きなメインビジュアルを配置する。
- メインビジュアルの下に「最新ニュース / お知らせ」を配置する。
- その下に、事業概要、研究開発、会社概要などの企業情報を縦に並べる。
- 全体として、1枚の長いスクロールページとして読める構成にする。

トップページに載せる想定セクション:
- メインビジュアル
- 最新ニュース
- 事業概要
- 研究開発の取り組み
- 会社概要


トップページ上の情報の扱い:
- 多くの情報は「それっぽさ」を出すための企業サイト風コンテンツでよい。
- ただし、最新ニュース欄には「新入社員紹介特設ページを追加しました」「当社の社内報が「社内報コンテスト」で入賞しました」などあわせて３つくらい。
- このお知らせをクリックすると、社内報の詳細ページまたはPDF表示ページへ遷移する。

5. ヘッダー / ナビゲーション要件
--------------------------------
ページ上部にはタブ状またはメニュー状のナビゲーションを配置する。

想定ナビゲーション:
- トップ
- お知らせ
- 事業内容
- 会社概要
- サイト内検索

ナビゲーションの方針:
- トップページ内のセクションへ移動するリンクと、別ページへ移動するリンクを混在させてもよい。
- 社員紹介など、トップページに載せたくない情報は、ニュースから別ページとして開けるようにする。
- 隠しページは通常ナビゲーションには表示しない。


8. 社員紹介 / 新入社員紹介要件
------------------------------
社員紹介ページは、トップページ内には入れず、別ページとして扱う。

検討中の方針:
- 通常の「社員紹介ページ」にする案。
- 「新入社員紹介特設ページ」または「新入社員紹介記事」にする案。
- 社内報の中に新入社員紹介を載せる案。

現時点の推奨:
- 「新入社員紹介特設ページ」または「新入社員紹介記事」として作る。
- 最新ニュースから「新入社員紹介特設ページを公開しました」という導線で遷移させる。
- 企業サイトとしては、通常の社員一覧よりも自然に見せやすい。

ページ内容:
- 3人から5人程度の社員または新入社員を掲載する。
- それぞれの紹介文は短めでよい。
- 謎解きに必要な場合のみ、名前、所属、趣味、ひとこと、などに手がかりを入れる予定。


9. サイト内検索要件
------------------
サイト内検索は、謎解き用の重要機能として実装する。

基本仕様:
- ヘッダーまたはページ上部に検索欄を配置する。
- 特定のキーワードを入力した場合のみ、通常のリンクからは到達できないページへ遷移できるようにする。
- 通常のキーワードでは、一般的な検索結果または「該当なし」を表示する。

隠しページの扱い:
- ナビゲーション、トップページ、ニュース一覧などからはリンクしない。
- 検索機能に特定ワードを入力したときだけ到達できる。
- GitHub Pages上の静的サイトとして作るため、検索処理はJavaScriptで実装するが、今後clouldworksやcloudpagesなども検討する。これによってargとしてネタバレしないサイト構造を実現する予定。

実装例:
- 検索結果ページを表示し、そこから隠しページへのリンクを出す。
- キーワードは完全一致（表記揺れ含む数パターンとの完全一致）

10. 隠しページ要件
-----------------
サイト内検索でのみ到達できるページを用意する。

役割:
- 参加者が特定のワードを発見した後の到達先。
- 次の謎、ログイン情報、外部アカウント情報、追加資料などを提示する。

注意点:
- URLを直接知られればアクセスは可能なため、完全な秘密情報は置かない。
- あくまで謎解き上の「発見した感」を作るための隠し導線として扱う。



12. デザイン要件
----------------
- 参考イメージは、製薬会社・ヘルスケア企業の公式サイト。
- 清潔感、信頼感、研究開発感を重視する。
- 白を基調に、信頼感のある青をアクセントカラーとして使う。
- メインビジュアルは研究室、製剤、植物由来成分、清潔な設備などのイメージが合う。
- 企業サイトとして自然に見えるよう、過度に謎解きっぽい演出は避ける。

13. アセット要件
----------------
現時点で必要になる可能性があるアセット:
- トップページ用メインビジュアル
- 社内報のPDFまたはPDF風デザイン
- 社員紹介用の人物写真またはシルエット
- ロゴ（八芒星に蜂をあしらったデザイン。企業名は八星製薬株式会社）
- 必要であれば、Xアカウント等へ誘導するための画像やアイコン

現在の方針:
- 足りない画像は仮画像または生成画像で代用する。
- 実在企業の画像やロゴは使用しない。
- 架空企業として成立するオリジナル素材を使う。

14. 未決事項
------------
今後決める必要がある項目:
- 社内報の号数、発行日、内容。
- 新入社員の人数、名前、所属、紹介文。
- サイト内検索で反応するキーワード。
- 検索で到達する隠しページの内容。
- Xアカウントなど外部サイトへの導線を実際に使うか。
- 社員専用ページ風の画面を作るか。
- PDFを本物のPDFとして作るか、HTMLで表現するか。

15. 次の実装ステップ
-------------------
1. トップページを現在の仮サイトから、ARG用の構成に合わせて調整する。
2. 最新ニュースに、社内報と新入社員紹介へのリンクを追加する。
3. 社内報ページをHTMLで作成する。
4. 新入社員紹介ページを作成する。
5. サイト内検索機能を実装する。
6. 特定キーワードで到達する隠しページを作成する。
7. 必要な手がかりを、ニュース、社内報、社員紹介、検索結果に配置する。
